## Our principles

> **Principle** (noun)
>
> A rule or guideline based on values that directs specific actions, decisions, or practices.
>
> **Purpose**: Provides actionable guidance to align with values in a consistent way.
>
> **Scope**: Specific to processes, workflows, or systems; more practical and directive.

We aim to build a design system based on consistency, accessibility, and flexibility. It provides reliable components, supports accessible design, and enables scalable customization to meet both product teams' and partners' needs.

## ‚öñÔ∏è **Consistent**

Build trust and predictability with a unified user experience through stable, reusable components and thoughtfully crafted guidelines.

### What does it mean?

- Predictable components ensure reliability and ease of use
- Changes to the system are purposeful and defendable, adhering to established principles over ad-hoc requests

### How have we implemented it?

- Consolidated Amount Field with clear and consistent functionality
- Form field component to control common input fields
- Icon and button sizes consistency

## üåç **Accessible**

Accessibility is foundational, ensuring the design system serves all users across devices, screen sizes, and needs.

### What does it mean?

- ADA-compliant by default
- Clear, focusable elements and sufficient contrast for all interactive components
- Education and mastery of accessibility within the team and organization

### How have we implemented it?

- Accessible color token system
- Accessible content design: descriptive headings and labels
- Consistent focus management across components like modals and forms

## **ü§∏ Flexible**

Balance flexibility and stability to empower teams to address unique needs while preserving the integrity of the design system.

### What does it mean?

- Future-friendly with a foundation-first approach.
- Supports partner theming while maintaining core system principles.

### How have we implemented it?

- Themeable token system for partner branding
- Slot components
- Badge component implementable on any component
