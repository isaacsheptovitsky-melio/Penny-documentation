## Usage

* By default, the menu is triggered when a user starts typing. This should be used when the content is readily familiar or commonplace enough to a user that they can begin populating values without seeing a list of all available options.
* If the content of the combo box is unfamiliar or complex, the menu should be triggered when focus is placed on the input field because a user would benefit from seeing example content before selecting a value.
* The Combobox can include predefined options to help users get started, but it can also function without them.
* It always requires a call to the server to fetch relevant options dynamically.
* After a selection is made:
    * The selected value moves to the background, indicating its selection.
    * The user cannot edit the selected value manually.
    * Users can either choose a new value or clear the current selection.

> **Note:** For details on sizes, behaviors, and general guidelines applicable to all form components, refer to the [Form Field](https://zeroheight.com/98bb1df01/p/6854b6-form-field) documentation.

> **Tip**: If server calls are unnecessary, consider using the 'select' component instead.

## Do's and don'ts

### ✅ Do
**Use a Combobox for long or dynamic lists**
Use a Combobox when users need to search within a long or dynamic list

![](/Penny-documentation/components/combobox/combobox-image-2.jpg)

### ❌ Don't
**Use Select for short, static lists**
Don't use a Combobox for short, static lists – use a 'select' component instead

## Desktop usage and treatment

Combobox components on desktop should provide clear visual feedback for typing states and dropdown interactions. Ensure the dropdown menu is properly positioned and sized to accommodate the content without obscuring other interface elements.

## Mobile usage and treatment

On mobile, the combobox transforms into a **full-screen mode** for an improved UX tailored to small screens.
