## Usage

* The component accepts files of specific formats and either displays a preview or not, depending on the file format
* Supported files:
    * CSV
    * JPEG (Preview)
    * JSON
    * PDF (Preview)
    * PNG (Preview)
    * SVG (Preview)
* After a file has been uploaded, the user is allowed to:
    * Delete uploaded file
    * Replace uploaded file

## Accessibility

Displaying the full filename isn't required to meet accessibility guidelines, but we have chosen to include it.

## Desktop usage and treatment

Large file upload components on desktop should provide clear visual feedback for drag-and-drop operations and support various file formats with appropriate preview capabilities. The interface should clearly communicate file size limitations and supported formats.

## Mobile usage and treatment

On mobile devices, large file upload components should adapt to touch interactions and consider the constraints of mobile file access. The preview functionality should be optimized for smaller screens while maintaining clear visual hierarchy.
