## Usage

* When users need to select one or multiple options from a list
* When a setting or preference can be toggled on or off
* When multiple selections are allowed independently (unlike radio buttons, which allow only one)
* When confirming agreement to terms and conditions or consent forms

## Desktop usage and treatment

Checkboxes on desktop should provide clear visual feedback for hover and focus states. They should be easily identifiable and maintain consistent spacing within forms or lists. The clickable area should extend to the label for better usability.

## Mobile usage and treatment

On mobile devices, checkboxes should have adequate touch target sizes (minimum 44px) and clear visual feedback for selection states. Consider the reduced screen space when positioning labels and ensure they remain easily readable.
