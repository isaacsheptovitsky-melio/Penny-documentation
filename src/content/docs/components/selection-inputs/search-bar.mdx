## Usage

* **Primary Use:** When users need to search for specific items in a large dataset or navigate content efficiently.
* **Placement:** Typically positioned at the top of a page, inside a navigation bar, or within a filtering panel.
* **Behavior:** Supports live search, autocomplete suggestions, and filtering options, depending on implementation.
* Allow users to clear the search input easily with a clear button (✖️)

> **Note:** For details on sizes, behaviors, and general guidelines applicable to all form components, refer to the [Form Field](https://zeroheight.com/98bb1df01/p/6854b6-form-field) documentation.

## Do's and don'ts

### ✅ Do
**Allow users to clear the search input easily**
Allow users to clear the search input easily with a clear button (✖️)

![](/Penny-documentation/components/search-bar/search-bar-image-1.png)

### ❌ Don't
**Don't reset user input unexpectedly**
Don't reset user input unexpectedly when navigating or refreshing the page.

![](/Penny-documentation/components/search-bar/search-bar-image-2.png)

## Desktop usage and treatment

Search bars on desktop should provide instant feedback for search queries and clear visual indicators for active states. Auto-complete and suggestions should be properly positioned and easily navigable with keyboard shortcuts.

## Mobile usage and treatment

On mobile devices, search bars should trigger appropriate keyboards (text input) and provide touch-friendly clear buttons. Consider using full-screen search overlays for better mobile user experience when dealing with complex search functionality.
