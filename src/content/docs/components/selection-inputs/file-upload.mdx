## Usage

Use the File upload component when:

* When users need to upload one file
* When drag-and-drop functionality enhances the experience
* When uploading a file that requires a user-visible preview, use the Large file uploader.
* After a file has been uploaded, the user is allowed to:
    * Delete uploaded file
    * Replace uploaded file

> **Note:** For details on sizes, behaviors, and general guidelines applicable to all form components, refer to the [Form Field](https://zeroheight.com/98bb1df01/p/6854b6-form-field) documentation.

## Desktop usage and treatment

File upload components on desktop should provide clear visual feedback for drag-and-drop operations, hover states, and loading progress. The drop zone should be clearly defined and provide appropriate visual cues for valid file types.

## Mobile usage and treatment

On mobile devices, file upload components should adapt to touch interactions and consider the limitations of mobile file access. The interface should clearly guide users through the upload process with appropriate progress indicators and error handling.
