## Usage

Use this component to visually represent a group of containers, stacked on top of one another.

## **Stacks To Display**

**Stacks to Display** controls the visual depth by adding "layers" behind the main container.

* **0:** A single, flat container with no background layers.
* **1:** Adds one visual layer behind the main container (creates a "double" stack effect).
* **2:** Adds two visual layers behind the main container (creates a "triple" stack effect).

## Padding

Adjust `paddingX` and `paddingY` independently to accommodate dense data grids (smaller padding) or airy marketing content (larger padding).

## Desktop usage and treatment

On desktop, stacked containers provide visual depth cues to indicate grouped content or hierarchical relationships. The stack effect should be subtle enough not to interfere with content readability.

## Mobile usage and treatment

On mobile, the stacked effect adapts to smaller screens while maintaining visual hierarchy. Consider reducing the stack depth on very small screens to avoid visual clutter.
