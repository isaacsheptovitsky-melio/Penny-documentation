The Virtual Card component is made out of the following atomic components:

- [Base Virtual Card](https://penny.melio.com/?path=/docs/data-display-components-virtual-card-base-virtual-card--docs)
- [Card number](https://penny.melio.com/?path=/docs/data-display-components-virtual-card-card-number--docs)
- [CVV](https://penny.melio.com/?path=/docs/data-display-components-virtual-card-cvv-number--docs)
- [Expiration date](https://penny.melio.com/?path=/docs/data-display-components-virtual-card-expiration-date--docs)
- [Details](https://penny.melio.com/?path=/docs/data-display-components-virtual-card-details--docs)

## Usage

Use this component to:

- Present the necessary details for a user or vendor to process a payment online
- Display the issuing network using the `brandType` prop
- Control the visibility of sensitive information (like the full card number and CVV) using the `showDetails` prop.
- show the specific value loaded onto the card or the remaining limit using the `amount` prop

## Desktop usage and treatment

Virtual cards should be prominently displayed with clear access to card details while maintaining security. Provide appropriate controls for showing/hiding sensitive information and ensure the card design is recognizable and professional.

## Mobile usage and treatment

On mobile devices, ensure virtual cards remain readable and functional. The card layout should adapt to smaller screens while maintaining the essential information hierarchy and security controls.
