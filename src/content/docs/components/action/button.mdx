## Usage

- Use for proper actions, such as submitting forms or initiating key flows
- Secondary and tertiary variants should support less prominent actions
- Use the Disabled variant to prevent interactions when an action is unavailable
  - However, don't use a disabled button to indicate that form submission would be invalid

### States

All buttons share the same set of states:

- **Rest**: the default appearance
- **Hover**: When the user points at the button with the cursor
- **Pressed**: The moment the button is clicked or tapped
- **Loading**: While the system processes the action
- **Disabled**: When the action is unavailable

## Left and right elements

The button includes two slots, left and right elements, typically used for icons that either clarify the button's purpose or suggest UI behavior.

![Left element](/Penny-documentation/components/button/button-left-element.png)
**Left element**

&nbsp;

![Right element](/Penny-documentation/components/button/button-right-element.png)
**Right element**

### Using both right and left elements

It's not recommended using both right and left elements too often, since it may clutter the UI and become repetitive, defeating the differentiating effect of the icon.

| ✅ **Do**                                                                                                                                                                              | ❌ **Don't**                                                                                                                                                                                    |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ![](/Penny-documentation/components/button/button-do-03.png) <br/> **Use the left element for global functions** <br/> Use for single global functions like a filter button on screen. | ![](/Penny-documentation/components/button/button-left-element-dont.png) <br/> **Avoid repeating left icons for similar actions** <br/> Repeating the same icon reduces its impact and clarity. |

### Additional use cases

Another common use of the left element is to display a brand symbol, such as in “Connect with Google” login buttons.

## Desktop usage and treatment

### Width

As a rule, buttons should hug their contents—meaning their width is defined by the content inside.
One exception to this rule is the Sign in button at the login screen.

![Hug content](/Penny-documentation/components/button/button-hug-content.png)
**Hug content**

&nbsp;

| ✅ **Do**                                                                                                          | ❌ **Don't**                                                                                                      |
| ------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------- |
| ![](/Penny-documentation/components/button/button-states.png) <br/> **Use hug contents to set the button's width** | ![](/Penny-documentation/components/button/button-layout.png) <br/> **Don't use fill container unless on mobile** |

### Placement

Place primary (essential) actions on the right and secondary or optional actions on the left.

![Button placement in a Bill details drawer](/Penny-documentation/components/button/button-placement-drawer.png)
**Button placement in a Bill details drawer**

&nbsp;

| ✅ **Do**                                                                                                                                                           | ❌ **Don't**                                                                                                                                                     |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ![Use hug contents to set the button's width](/Penny-documentation/components/button/button-width-hug.png) <br/> **Place buttons horizontally, next to each other** | ![Don't use fill container](/Penny-documentation/components/button/button-width-fill-dont.png) <br/> **Don't place buttons vertically, one on top of the other** |

| ✅ **Do**                                                                                                                                                                      | ❌ **Don't**                                                                                                                                                                                            |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ![](/Penny-documentation/components/button/button-do-01.png) <br/> **Expose up to 3 buttons** <br/> Limit to maximum 3 buttons per screen section to avoid overwhelming users. | ![](/Penny-documentation/components/button/button-dont-01.png) <br/> **Don't expose more than 3 buttons for a part of the screen** <br/> Too many buttons create decision paralysis and visual clutter. |

| ✅ **Do**                                                                                                                               | ❌ **Don't**                                                                                                                                    |
| --------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| ![](/Penny-documentation/components/button/button-placement-do.png) <br/> **Place the essential action on the far-right of the layout** | ![](/Penny-documentation/components/button/button-placement-dont.png) <br/> **Don't place secondary buttons to the right of prominent buttons** |

<Aside type="tip">
  These guidelines support consistent action arrangement. However, if an
  optional action needs extra emphasis due to its business impact, it's okay to
  break the rule.
</Aside>

## Mobile usage and treatment

### Width

On mobile devices, it’s best to place buttons side by side, ideally using a full-width container to give them equal widths.

![Fill container on mobile](/Penny-documentation/components/button/button-mobile-fill-container.png)
**Fill container on mobile**

### Vertical placement

If the button content doesn’t fit side by side, stack the buttons vertically instead.
Place the primary button at the bottom to make it easier to reach with the thumb.

![Vertical placement](/Penny-documentation/components/button/button-mobile-vertical-placement.png)
**Vertical placement**
