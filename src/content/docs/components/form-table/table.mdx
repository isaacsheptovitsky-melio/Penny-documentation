## Usage

- Use for presenting structured, comparable information 
- Display data that users need to scan, sort, or filter
- Organize complex datasets into digestible formats
- Enable bulk actions on multiple data entries

## Patterns of use

We have different patterns of use for the table, where we use different features to achieve the best UX for our users.

### Pinned columns

Pinned columns stay fixed to one side of the table when it scrolls horizontally, ensuring they remain visible to the user.

### Pagination

Pagination divides content into multiple pages, helping prevent excessively long scrolling and improving performance for large datasets.

### Sticky header

Sticky headers remain visible when users scroll through long tables, providing constant reference to column labels.

## Content guidelines

### Data formatting

- Use consistent formatting for similar data types
- Align numerical data appropriately for easy comparison
- Provide clear, descriptive column headers
- Use appropriate date and time formats

### Text treatment

- Keep cell content concise and scannable
- Use truncation with tooltips for long content
- Maintain consistent capitalization and terminology

## Desktop usage and treatment

### Layout and spacing

- Use appropriate column widths for content types
- Maintain consistent row heights for visual rhythm
- Provide adequate padding within cells
- Consider horizontal scrolling for wide tables

### Interaction patterns

- Enable sorting on relevant columns
- Provide clear selection mechanisms
- Support keyboard navigation
- Use hover states to indicate interactive elements

## Mobile usage and treatment

### Responsive strategies

- Consider card layouts for complex data on small screens
- Hide less critical columns on narrow viewports
- Maintain essential data visibility
- Use horizontal scrolling strategically for key columns
- Ensure touch targets meet accessibility requirements
