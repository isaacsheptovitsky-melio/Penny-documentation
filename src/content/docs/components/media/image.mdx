## Usage

- Use to display images in various formats and aspect ratios
- Configure aspect ratios to maintain consistent visual hierarchy
- Apply border radius to match design system standards
- Use object fit properties to control how images fill their containers
- Leverage fallback functionality for improved user experience when images fail to load

## States

### Disabled

Lowers opacity to indicate the image or associated action is unavailable.

## Props

### Aspect ratio

Supports standard ratios:

- 1:1
- 4:3
- 16:9
- 7:3 

### Border Radius

Offers multiple rounding options ranging from `none` to `full`:

- `none`: 0px
- `50`: 2px
- `100`: 4px
- `200`: 8px
- `300`: 12px
- `400`: 16px
- `600`: 24px
- `full`: 9999px

### Object Fit

Control how the image fills its container using `cover` (fills the area), or `contain` (shows the whole image).

## Fallback

Automatically displays a placeholder if the image source is broken.

### Fallback Icon

Allows for a specific icon (like a "File" or "Warning" icon) to appear on load error.

## Desktop usage and treatment

- Use appropriate aspect ratios for content type and layout requirements
- Apply consistent border radius values across similar UI elements
- Consider object fit settings based on content importance and layout constraints
- Ensure fallback icons are accessible and clearly communicate the error state

## Mobile usage and treatment

- Maintain aspect ratios for consistency across device sizes
- Ensure images remain accessible and properly scaled on smaller screens
- Consider bandwidth limitations when selecting image quality and formats
- Verify fallback states work effectively on touch interfaces
